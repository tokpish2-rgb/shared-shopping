# Общая закупка

Мобильный статический сайт для совместных покупок: админ создает список, приглашает людей по ссылке или QR-коду, участники добавляют позиции и разбирают количество между своими корзинами.

## Что уже есть

- создание списка админом;
- приглашение по ссылке и QR-коду;
- участники без регистрации;
- общий список и личные корзины;
- деление количества, например `вода 10 шт.` → `Маша 4`, `Петя 6`;
- отметка `куплено`;
- локальный режим без сервера;
- заготовка синхронизации через Firebase Realtime Database.

## Запуск

Откройте `index.html` в браузере или положите файлы в GitHub Pages.

Без Firebase данные сохраняются только в браузере текущего устройства. Для настоящей совместной работы нужен `config.js`.

## Настройка синхронизации

1. Создайте проект Firebase.
2. Включите Authentication → Anonymous.
3. Создайте Realtime Database.
4. Скопируйте `config.example.js` в `config.js`.
5. Вставьте Firebase web app config:

```js
window.SHOPSYNC_FIREBASE_CONFIG = {
  apiKey: "...",
  authDomain: "...",
  databaseURL: "...",
  projectId: "...",
  storageBucket: "...",
  messagingSenderId: "...",
  appId: "..."
};
```

6. В `index.html` замените:

```html
<script src="config.example.js"></script>
```

на:

```html
<script src="config.js"></script>
```

## Черновые правила Firebase

Для MVP можно начать с таких правил. Они требуют анонимного входа, разрешают участникам читать список и менять покупки, а удаление участников/настроек лучше потом ужесточить отдельными правилами.

```json
{
  "rules": {
    "lists": {
      "$listId": {
        ".read": "auth != null",
        ".write": "auth != null"
      }
    }
  }
}
```

Перед публичным запуском правила нужно усилить: админские операции разрешать только `adminUid`, а участникам оставить добавление позиций и редактирование своих назначений.
